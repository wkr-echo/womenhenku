# v2 功能详细说明

> 本文件描述 v2 三项功能（⑥标签系统、⑦Token 用量统计、⑧多语言 UI）的完整行为规范。
> 开发按 v1 五功能完成后再追加，但上交版本需包含此三项。
> 来源：原始 Mercury 代码分析。

---

## ⑥ 标签系统

标签系统分三个子功能：单篇打标签、批量打标签、标签库管理。

### A. 单篇文章手动打标签

**入口**：阅读页工具栏 `#` 按钮 → 弹出浮动面板（宽度 280pt）。

**面板布局**：

```
┌─────────────────────────────────┐
│  Tags                           │
│  ┌─────────────────────────┐    │
│  │ 输入标签（逗号分隔）      │    │  ← 自由文本输入框
│  │ [AI, 机器学习]           │    │
│  └─────────────────────────┘    │
│  [添加] 按钮（输入为空时禁用）    │
├─────────────────────────────────┤
│  Suggested                     │
│  [OpenAI] [Rust] [Python]      │  ← NLP 自动提取（最多 5 个）
│  [RSS] [跨平台]                │  ← AI 建议（异步加载）
│  Generating suggestions... ◌   │  ← AI 加载中
├─────────────────────────────────┤
│  Did you mean: [已有的标签名]    │  ← 拼写纠错提示
├─────────────────────────────────┤
│  Existing                      │
│  [Rust] [Tauri] [React]        │  ← 输入时前缀匹配（最多 10 个）
├─────────────────────────────────┤
│  已应用：                       │
│  [AI ×] [编程 ×]                │  ← 每行可删除
│  No tags yet                   │  ← 无标签时占位
└─────────────────────────────────┘
```

**完整交互行为**：

| 行为 | 细节 |
|---|---|
| 添加标签 | 输入框键入 → 按回车或点 [添加] → 按逗号/换行分割 → 去除空白 → 丢弃空 token → 逐条 assign |
| 删除标签 | 点击标签右侧 `×` 按钮 → 从该文章移除此标签 |
| NLP 建议 | 面板出现时立即从文章标题+摘要提取实体（组织名/人名/地名），同步返回，不请求网络 |
| AI 建议 | 面板出现时自动触发（需 `Agent.Tagging.Enabled=true` + Provider 可用），异步加载，结果合并到 NLP 建议前面。显示 "Generating suggestions..." spinner。超时 60 秒 |
| 已有标签自动补全 | 空闲时显示热门正式标签（最多 10 个）；输入时搜索所有标签（含临时标签），前缀匹配 |
| 拼写纠错 | 输入空格或逗号后对最后一个 token 进行纠错。精确匹配 → 无建议；模糊匹配（编辑距离 ≤ 2）→ 显示 "Did you mean: ..."；全大写/CamelCase 词跳过纠错 |
| 离开面板 | 关闭面板时自动取消 AI 请求；Reader 管线重建时也取消 |
| 重复打开 | 同文章重新打开面板 → 取消上一次的 AI 请求 → 重新触发 |

**AI 建议不可用时的降级**：

| 原因 | 面板显示 |
|---|---|
| 未启用（Agent.Tagging.Enabled=false） | 仅显示 NLP 建议 |
| Reader 管线正在重建 | "AI suggestions unavailable while reader content is refreshing." |
| 无可用 Provider | 仅显示 NLP 建议，无错误提示 |
| LLM 超时/失败 | NLP 建议保留，错误显示在 Reader Banner |

**标签来源标记**：每次 assign 记录来源——面板手动添加标记 `"manual"`。

### B. 批量打标签

**入口**：设置 → Batch Tagging。

**完整流程（五阶段状态机）**：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ configure │ → │ running  │ → │readyNext │ → │  review  │ → │ applying │ → done
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     │               │                                                  │
     └── 可取消 ──────┴──────────────── 可取消 ─────────────────────────┘
```

**1. configure 阶段（配置）**

| 配置项 | 选项 | 默认值 |
|---|---|---|
| 范围选择 | 1周/1月/3月/半年/1年/全部/未读 | — |
| 跳过已批量打标签的文章 | 开关 | true |
| 跳过已有标签的文章 | 开关 | true |
| 并发度 | 滑块 1~5 | 3 |

- 进入 configure 时自动估算候选文章数
- 候选数 ≤ 0：底部显示 "no eligible entries"，开始按钮禁用
- 候选数 > 100：底部显示黄色警告 "超过 100 篇"
- 候选数 > 2000：**阻止启动**，底部显示红色 "超过 2000 篇安全上限"

**2. running 阶段（运行中）**

| 元素 | 行为 |
|---|---|
| 进度条 | `processed / total`，实时更新 |
| 计数 | 成功 / 失败分开计数 |
| 中止 | [Stop] 按钮 → 取消所有进行中任务，状态 → cancelled |
| 配置锁定 | 运行中禁止修改配置（`locksConfiguration = true`） |

- 每篇文章并发调用 LLM，失败自动重试（最多 3 次，指数退避 1s→12s）
- 已删除文章（isDeleted=true）自动跳过
- LLM 结果中每篇最多取 5 个标签、最多 2 个新标签提案

**3. readyNext 阶段（准备就绪）**

- 显示摘要：已处理 / 成功 / 失败 / 建议标签数
- 如果有新标签提案（newTagCount > 0）→ 进入 review
- 如果没有新标签提案 → 可直接进入 apply

**4. review 阶段（审查）**

```
┌──────────────────────────────────────────────────┐
│  以下标签由 AI 提出，请确认是否保留：              │
│                                                  │
│  标签名        命中次数    涉及文章数    决定       │
│  ───────────────────────────────────────────     │
│  AI             15         12           [保留 ▾] │
│  机器学习        8          8           [丢弃 ▾] │
│  深度学习        3          3           [保留 ▾] │
│                                                  │
│  [全部保留]  [全部丢弃]                           │
│                                                  │
│  有 2 个审查决定待处理              [应用] 按钮    │
└──────────────────────────────────────────────────┘
```

- 每个新标签显示：名称、命中次数、涉及文章数
- 每个标签决定：保留 / 丢弃（默认 pending）
- "全部保留" / "全部丢弃" 批量操作
- 有 pending 决定时 [应用] 按钮禁用
- 丢弃后不可恢复

**5. applying 阶段（应用）**

- 每 50 篇分块写入数据库
- 进度条实时更新
- 完成弹窗显示最终统计：

```
批量标签完成
- 已处理：120 篇
- 成功：115 篇
- 失败：5 篇
- 添加标签关联：320 条
- 创建新标签：3 个
- 保留提案：2 个
- 丢弃提案：1 个
```

**取消和丢弃**：
- running/review/applying 阶段可取消 → 状态变为 cancelled
- cancelled 状态可重置重新开始
- 丢弃仅在不运行时可用 → 清除所有暂存数据

**安全约束汇总**：

| 约束 | 值 |
|---|---|
| 警告阈值 | 100 篇 |
| 绝对上限 | 2000 篇 |
| 每篇最多标签 | 5 个 |
| 每篇最多新提案 | 2 个 |
| LLM 词汇注入上限 | 50 个 |
| 并发度 | 3（可调 1~5） |
| 应用分块大小 | 50 篇 |
| 失败重试次数 | 3 次 |
| 重试延迟 | 1s 起，上限 12s |

### C. 标签库管理

**入口**：设置 → Tag Library。

**布局**：左侧主列表 + 右侧检查器。

```
┌──────────────────────┬──────────────────────────┐
│ 搜索：[________]     │  标签名：Rust             │
│ 过滤：[全部 ▾]       │  使用次数：42             │
│                      │  状态：正式               │
│  Rust (42)           │  创建时间：2026-01-15     │
│  Tauri (35)          │  别名：rust-lang         │
│  React (28)          │                          │
│  AI (15) 临时        │  [重命名] [添加别名]      │
│  Python (0) 未使用    │  [设为永久] [合并]       │
│                      │  [删除]                   │
│                      │                          │
│                      │  潜在重复：               │
│                      │  • Rust (42)             │
│                      │  • rust (3)              │
│                      │  [合并到 Rust]           │
└──────────────────────┴──────────────────────────┘
```

**过滤选项**：

| 过滤 | 显示 |
|---|---|
| 全部 | 所有标签，按使用次数降序 → 名称字母序 |
| 临时 | `usageCount < 2` 的标签 |
| 未使用 | `usageCount = 0` 的标签 |
| 有别名 | 至少有一个别名的标签 |
| 潜在重复 | 被检测为重复候选的标签 |

**检查器操作**：

| 操作 | 行为 | 验证 |
|---|---|---|
| **重命名** | 修改标签名 | 非空、不与已有标签冲突 |
| **添加别名** | 给标签加一个同义词 | 非空、不与标准名相同、不与已有别名冲突 |
| **删除别名** | 移除某个别名 | — |
| **设为永久** | 将临时标签转为正式 | — |
| **合并** | 将 A 合并到 B：A 的 entry_tag 全转到 B，A 的兼容别名转到 B，A 的名字作为 B 的别名，删除 A | A ≠ B、批量标签运行中禁止 |
| **删除** | 级联删除 entry_tag、tag_alias、tag | 批量标签运行中禁止 |
| **删除未使用** | 批量删除 `usageCount=0` 的标签 | 批量标签运行中禁止 |

**操作结果反馈**：

| 类型 | 颜色 | 示例 |
|---|---|---|
| 成功 | 绿色 | "Tag renamed." |
| 警告 | 黄色 | "No unused tags to delete." |
| 错误 | 红色 | "This name already exists." |

**批量标签运行期间的锁定**：所有标签库的破坏性操作（重命名/合并/删除/设为永久/删除未使用）在批量标签活跃期间禁止，显示红色提示 "Tag library mutations are unavailable while batch tagging is active."

### D. 标签规范化规则

所有标签写入路径必须经过统一规范化：

```
输入 → 去除首尾空白 → 转小写 → 折叠分隔符（-_. 和空格折叠为单个空格）
```

| 输入 | 规范化结果 |
|---|---|
| `"  AI  "` | `"ai"` |
| `"Machine-Learning"` | `"machine learning"` |
| `"foo...bar"` | `"foo bar"` |
| `"hello   world"` | `"hello world"` |
| `"   "` | `""`（拒绝） |

---

### E. 标签数据模型

| 表 | 关键字段 |
|---|---|
| `tags` | id, name, normalizedName, isProvisional, usageCount |
| `tag_alias` | id, tagId, alias, normalizedAlias |
| `entry_tag` | entryId, tagId, source("manual"/"rss"/"batch"), confidence |

**临时标签规则**：新建标签 `isProvisional=true`。当 `usageCount ≥ 2` 时自动升为正式。删除时 `usageCount` 降到 2 以下则降回临时。

---

### F. 重复标签检测

三种检测算法（优先级从高到低）：

| 类型 | 规则 | 示例 |
|---|---|---|
| 复数变体 | 去除复数后缀后 token 相同 | "Databases" ↔ "Database" |
| 命名变体 | 去空格后编辑距离 ≤ 1，长度 ≥ 6 | "MacOS" ↔ "Mac OS" |
| 拼写变体 | 单词长度 ≥ 5，首尾字母相同，编辑距离 ≤ 2 | "Pytohn" ↔ "Python" |

已在别名中消解的标签对不显示为重复候选。

## ⑦ Token 用量统计

### 数据来源

每次 Summary/Translation 调用 LLM 后自动记录一条 `LLMUsageEvent`：

| 记录字段 | 说明 |
|---|---|
| Provider 快照 | baseURL、名称、host（即使 Provider 被删除也不影响历史报表） |
| Model 快照 | model ID、名称 |
| Agent 类型 | summary / translation |
| Token 数 | prompt_tokens、completion_tokens、total_tokens |
| 请求状态 | 成功 / 失败 / 取消 / 超时 |
| 时间戳 | 精确到秒 |

### 报表类型

**A. 单对象统计**

```
时段: [1周 | 2周 | 1月]    任务: [全部 | 摘要 | 翻译]

┌─────────────────────────────────────┐
│  每日 Token 用量                     │
│  ▁▂▃▅▆█▇▅▄▃▂▁▂▃▄  ← 折线图          │
│  鼠标悬停显示当天详情                  │
├─────────────────────────────────────┤
│  总计         │  12,345 tokens       │
│  提示 token   │  5,678              │
│  补全 token   │  6,667              │
│  请求数       │  42 次              │
│  成功率       │  97.6%              │
│  平均/请求    │  294 tokens         │
├─────────────────────────────────────┤
│  较上期：Token ↑ 15%  请求 ↑ 8%      │  ← 绿色=上升，红色=下降
└─────────────────────────────────────┘
```

**B. 对比报表**

所有 Provider / Model / Agent 按用量降序排列的水平条形图，可切换指标（Tokens / 输入 / 输出 / 请求数）。

### 设置

- **数据保留策略**：1月 / 3月 / 6月 / 1年 / 永久（默认 6 个月）
- 启动时自动清理过期数据
- 可手动「清除过期」或「清除全部」（确认弹窗）

## ⑧ 多语言 UI

### 支持范围

| | UI 语言 | Agent 输出语言 |
|---|---|---|
| 控制什么 | 按钮、菜单、提示文字 | 摘要/翻译的输出语言 |
| 可选数量 | **2 种**（中文、英文） | **12 种**（中/英/日/韩/法/德/西/葡/俄/阿/印地/繁体中文） |
| 切换方式 | 设置 > 通用 > 语言下拉框 | 设置 > Agent > 目标语言下拉框 |
| 生效方式 | **即时生效，无需重启** | 仅影响后续新生成的摘要/翻译 |
| 默认值 | 跟随操作系统语言 | 中文（我们自定义，原版默认英文） |

### 实现约束

- 所有 UI 字符串用 `t()` 函数包裹，从语言资源文件读取
- 语言资源文件：`src-ui/src/locales/zh.json`、`en.json`
- 不允许硬编码任何面向用户的字符串
- 调试/错误日志不翻译（保持英文）

### 12 种 Agent 输出语言

zh-Hans（简体中文）、zh-Hant（繁体中文）、en（英文）、ja（日文）、ko（韩文）、fr（法文）、de（德文）、es（西班牙文）、pt-BR（葡萄牙文-巴西）、ru（俄文）、ar（阿拉伯文）、hi（印地文）
