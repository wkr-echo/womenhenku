# Mercury

A cross-platform RSS reader with AI-powered features for comfortable information aggregation and reading experience.

## Features

### ① Core Features: Feed / OPML Parsing + Sync + Content Presentation

- RSS / Atom / JSON Feed subscription and parsing
  - Support RSS 0.91/0.92/1.0/2.0, Atom 1.0, JSON Feed formats
  - Add, delete, refresh feed management
- OPML import and export
  - Batch import/export feeds
  - Feed grouping and organization
- Feed sync
  - Auto sync with configurable intervals
  - Manual sync trigger
  - Sync status display
- Content presentation
  - Article list (read/unread status, search, pagination)
  - Article reading mode
  - Sidebar feed count

### ② Content Cleaning: Cleaned HTML + Cleaned Markdown + Custom Styles

- Cleaned HTML
  - Built-in Readability algorithm for content extraction
  - HTML tag cleaning and normalization
  - Remove ads, navigation and irrelevant elements
- Cleaned Markdown
  - HTML to Markdown conversion
  - GFM support (tables, task lists, code blocks)
- Custom styles
  - Theme switching (light/dark/custom)
  - Font customization
  - Reading mode layout optimization

### ③ AI Feature 1: Summary Agent + LLM Providers

- Summary Agent
  - Article summarization
  - Configurable summary language
  - Configurable summary detail level
  - Streaming output for real-time results
  - Customizable prompt templates
- LLM Providers
  - OpenAI-compatible protocol implementation
  - Support OpenAI, DeepSeek, Ollama, vLLM, etc.
  - Provider/model configuration management
  - Secure API Key storage

### ④ AI Feature 2: Translation Agent

- Translation Agent
  - Paragraph-level bilingual translation
  - Bilingual comparison layout
  - Concurrent paragraph translation
  - Retry failed paragraphs
  - Clear translation results
  - HY-MT2 optimization mode

### ⑤ Auxiliary Features: Multi-language Support, Logging and Debugging Tools

- Multi-language support
  - UI language switching (Chinese/English)
  - Localization resource management
- Logging
  - Application runtime logs
  - Error log collection
  - Debug information output
- Debugging tools
  - Development mode debug panel
  - API call logs
  - Database query logs

### ⑥ Auxiliary Features: LLM Usage Statistics

- Token usage statistics
  - Provider-level statistics
  - Model-level statistics
  - Agent-level statistics
- Comparison reports
  - Usage trend charts
  - Cost estimation
  - Historical records query

### ⑦ Notes and Digest Export: Notes + Single Export + Multi Export

- Notes functionality
  - Single article Markdown notes
  - Notes editing and saving
- Single export
  - Markdown format export
  - HTML format export
- Multi export
  - Digest combination export
  - Custom export templates
  - Share via system sharing service

### ⑧ Tag System: Article Tags + Filter by Tags + Tag Agent + Tag Management

- Article tags
  - Manual tag addition
  - Filter articles by tags
- Tag Agent
  - AI tag suggestions
  - Batch tagging
- Tag management
  - Tag library maintenance
  - Tag merging
  - Tag aliasing
  - Tag cleaning

## Technical Constraints

### ① Product Experience: Good design standards, elegant and simple user experience

- Follow modern UI design standards
- Clean and intuitive interaction flow
- Responsive layout for different screen sizes
- Smooth animations and transitions
- Reasonable information hierarchy and visual feedback

### ② Local-first: No registration/login or subscription required, never actively collect user data

- All data stored in local SQLite database
- No account registration or login required
- Never actively collect user personal data
- Never send user reading habits to cloud
- Privacy protection first

### ③ Platform-neutral: Support Windows / Linux / macOS

- Windows 10+ (WebView2 runtime)
- macOS 12+
- Linux (Wayland + X11, WebViewGTK)
- Unified codebase supporting multiple platforms
- Native installer formats for each platform

### ④ LLM-neutral: Support any LLM service providing standard API (including local)

- OpenAI-compatible protocol implementation (`/v1/chat/completions`)
- User-configurable baseURL, API Key, model name
- Support local LLMs (Ollama, vLLM, etc.)
- Not tied to specific LLM provider
- Streaming output support

### ⑤ Coding Agent Traceability: Form persistent, valuable work process documentation

- Record key technical decisions
- Maintain AGENTS.md for constraints and rules
- Maintain PLAN.md for execution plans
- Preserve design documents and technical notes
- Improve code comments and documentation

### ⑥ Team Collaboration Traceability: Correctly record committers and commit history

- Use Git version control
- Clear commit messages

- Pull Request workflow

## Developing

### Technical notes

- Backend and frontend separated architecture
  - Rust backend handles business logic and data persistence
  - React frontend handles UI rendering and user interaction
  - Tauri IPC for communication between backend and frontend
- Database design inspired by original Mercury schema
  - SQLite with WAL mode for concurrent access
  - Versioned migrations for schema evolution
- AI protocol design inspired by OpenAI API
  - `/v1/chat/completions` compatible endpoint
  - Streaming support using Tauri Events
- Feed parsing using `feed-rs` crate
- HTML cleaning using `scraper` crate
- Markdown rendering using `comrak` crate

### Environment

- Rust 1.75+
- Node.js 18+
- Tauri CLI 2.x
- Tauri 2 + React + TypeScript
- SQLite (`rusqlite`)
- `reqwest` HTTP client
- `feed-rs` Feed parsing
- `scraper` HTML cleaning
- `comrak` Markdown rendering
- Vite frontend build

### Cross-platform compatibility

- Windows 10+ (WebView2 runtime built-in)
- macOS 12+
- Linux (Wayland + X11, WebViewGTK dependency)
- Path handling using Rust `dirs` crate
- Credential storage using Tauri secure-store plugin
