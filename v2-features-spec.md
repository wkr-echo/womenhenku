## ⑥ 标签系统

标签系统分三个子功能：单篇打标签、批量打标签、标签库管理。

### A. 单篇文章手动打标签

**入口**：阅读页工具栏 `#` 按钮 → 弹出浮动面板。

**面板内容**：

```
┌─────────────────────────────┐
│  输入标签（逗号分隔）         │  ← 自由文本输入
│  [AI, 机器学习, 编程]         │
├─────────────────────────────┤
│  建议（NLP 自动提取）：        │
│  [OpenAI] [Rust] [Python]   │  ← 从文章中提取的实体
│                             │
│  建议（AI 推荐）：            │
│  [RSS] [跨平台]              │  ← LLM 分析后推荐
├─────────────────────────────┤
│  已有标签：                   │
│  [Rust] [Tauri] [React]     │  ← 自动补全
├─────────────────────────────┤
│  已应用：                     │
│  [AI ×] [编程 ×]             │  ← 每行可删除
└─────────────────────────────┘
```

**交互行为**：
- NLP 建议**立即显示**（从文章标题和摘要提取组织名/人名/地名）
- AI 建议**异步加载**（需配置 AI Provider，面板显示加载动画）
- 输入时实时拼写纠错：接近已有标签时提示 "Did you mean: ..."
- 已有标签根据输入前缀自动过滤
- 最多显示 5 个建议、10 个已有标签
- 切换文章或关闭面板时取消 AI 请求

### B. 批量打标签

**入口**：设置 → 批量标签。

**流程**：

```
1. 配置阶段：
   - 选择范围（1周/1月/3月/半年/1年/全部/未读）
   - 开关：跳过已有标签的文章
   - 并发度滑块（1~5）
   - 显示预估文章数

2. 运行阶段：
   - 进度条（已处理/成功/失败计数）
   - 可中止

3. 审查阶段（AI 提出新标签时）：
   - 每个新标签显示：名称、出现次数、涉及文章数
   - 用户选择：保留 / 丢弃（支持"全部保留""全部丢弃"）
   - 审查完成后点击"应用"

4. 应用阶段：
   - 后台批量写入（每 50 篇分块）
   - 完成弹窗显示最终统计
```

**安全约束**：
- 超过 100 篇显示警告，超过 2000 篇阻止启动
- 每篇最多 5 个标签
- 批量并发度 3
- 失败重试最多 3 次

### C. 标签库管理

**入口**：设置 → 标签库。

**布局**：左侧列表 + 右侧检查器。

```
┌──────────────────┬──────────────────────┐
│ 搜索：[____]     │  标签名：Rust         │
│ 过滤：全部/临时   │  使用次数：42         │
│                  │  状态：正式           │
│  Rust (42)       │  创建时间：2026-01-15 │
│  Tauri (35)      │  别名：rust-lang     │
│  React (28)      │                      │
│  AI (15) 临时    │  [重命名] [添加别名]   │
│  Python (0) 未用  │  [合并] [删除]        │
└──────────────────┴──────────────────────┘
```

**操作**：重命名、添加别名、删除别名、合并到另一个标签、设为永久、删除、批量删除未使用标签。

**规范化规则**：所有标签统一转为小写、多空格折叠为单空格。通过别名表解析同义词。

**临时标签**：使用次数 < 2 为临时，≥ 2 自动升为正式（手动添加和面板接受的标签）。批量标签始终保持临时直到用户审查确认。

## ⑦ Token 用量统计

### 数据来源

每次 Summary/Translation 调用 LLM 后自动记录一条 `LLMUsageEvent`：

| 记录字段 | 说明 |
|---|---|
| Provider 快照 | baseURL、名称、host（即使 Provider 被删除也不影响历史报表） |
| Model 快照 | model ID、名称 |
| Agent 类型 | summary / translation |
| Token 数 | prompt_tokens、completion_tokens、total_tokens |
| 请求状态 | 成功 / 失败 / 取消 / 超时 |
| 时间戳 | 精确到秒 |

### 报表类型

**A. 单对象统计**

```
时段: [1周 | 2周 | 1月]    任务: [全部 | 摘要 | 翻译]

┌─────────────────────────────────────┐
│  每日 Token 用量                     │
│  ▁▂▃▅▆█▇▅▄▃▂▁▂▃▄  ← 折线图          │
│  鼠标悬停显示当天详情                  │
├─────────────────────────────────────┤
│  总计         │  12,345 tokens       │
│  提示 token   │  5,678              │
│  补全 token   │  6,667              │
│  请求数       │  42 次              │
│  成功率       │  97.6%              │
│  平均/请求    │  294 tokens         │
├─────────────────────────────────────┤
│  较上期：Token ↑ 15%  请求 ↑ 8%      │  ← 绿色=上升，红色=下降
└─────────────────────────────────────┘
```

**B. 对比报表**

所有 Provider / Model / Agent 按用量降序排列的水平条形图，可切换指标（Tokens / 输入 / 输出 / 请求数）。

### 设置

- **数据保留策略**：1月 / 3月 / 6月 / 1年 / 永久（默认 6 个月）
- 启动时自动清理过期数据
- 可手动「清除过期」或「清除全部」（确认弹窗）

## ⑧ 多语言 UI

### 支持范围

| | UI 语言 | Agent 输出语言 |
|---|---|---|
| 控制什么 | 按钮、菜单、提示文字 | 摘要/翻译的输出语言 |
| 可选数量 | **2 种**（中文、英文） | **12 种**（中/英/日/韩/法/德/西/葡/俄/阿/印地/繁体中文） |
| 切换方式 | 设置 > 通用 > 语言下拉框 | 设置 > Agent > 目标语言下拉框 |
| 生效方式 | **即时生效，无需重启** | 仅影响后续新生成的摘要/翻译 |
| 默认值 | 跟随操作系统语言 | 中文（我们自定义，原版默认英文） |

### 实现约束

- 所有 UI 字符串用 `t()` 函数包裹，从语言资源文件读取
- 语言资源文件：`src-ui/src/locales/zh.json`、`en.json`
- 不允许硬编码任何面向用户的字符串
- 调试/错误日志不翻译（保持英文）

### 12 种 Agent 输出语言

zh-Hans（简体中文）、zh-Hant（繁体中文）、en（英文）、ja（日文）、ko（韩文）、fr（法文）、de（德文）、es（西班牙文）、pt-BR（葡萄牙文-巴西）、ru（俄文）、ar（阿拉伯文）、hi（印地文）
