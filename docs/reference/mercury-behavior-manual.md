# Mercury 行为手册

> 本文件描述 Mercury 原版的用户功能行为与交互流程。
> 复刻版的 UI 实现可以不同，但用户看到什么、能做什么、交互流程必须与此一致。
> 来源：INIT.md 功能描述 + 原始 Mercury AGENTS.md 合约规范。

---

## ① Feed 订阅与内容呈现

### 用户能做什么

- **添加订阅源**：输入 URL → 应用自动抓取并解析 Feed → 显示 Feed 标题和文章列表
- **删除订阅源**：从侧边栏删除，同时清除该源的所有文章和内容
- **刷新**：手动刷新单个 Feed 或全部 Feed
- **自动同步**：应用按配置间隔（默认 30 分钟）自动检查新文章
- **OPML 导入**：选择 OPML 文件 → 批量添加订阅源。部分失败不中断（成功的添加，失败的提醒）
- **OPML 导出**：导出当前所有订阅源为 OPML 文件

### 用户看到什么

- **侧边栏**：订阅源列表，每项显示 Feed 标题 + 未读计数
- **文章列表**：选中 Feed 后显示文章列表，含标题、作者、发布时间
- **已读/未读**：未读文章标题加粗，已读文章正常字重
- **分页**：列表底部加载更多
- **搜索**（Stage 2）：搜索框输入关键词 → 搜索标题 + 摘要 → 结果列表

### 交互流程

```
侧边栏点击 Feed → 文章列表加载 → 点击文章 → 进入阅读
                                          → 标记已读
```

## ② 文章清洗与阅读体验

### 用户能做什么

- **阅读文章**：打开文章后看到清洗渲染后的内容
- **切换主题**：亮色 / 暗色模式，即时生效，重启保持
- **切换字体**：从系统字体列表中选择，阅读区即时变化
- **离线阅读**：已加载的文章断网后仍可阅读（缓存至 SQLite）

### 用户看到什么

- **清洗后的内容**：文章正文（去除导航、广告、侧边栏），支持标题、段落、列表、链接、图片、表格、代码块
- **渲染格式**：GFM Markdown 渲染——表格有边框、代码块有语法高亮、任务列表可勾选

### 交互流程

```
打开文章 → 自动执行 Readability 提取 → 清洗 → Markdown 转换 → 渲染展示
                                                              → 失败时展示原始 HTML
```

## ③ AI 摘要

### 用户能做什么

- **触发摘要**：在阅读文章中点击摘要按钮
- **配置选项**：选择目标语言（中文/英文等）、详细程度（简短/标准/详细）
- **流式展示**：摘要内容逐字流式输出，实时可见
- **取消**：摘要生成过程中可取消
- **查看历史**：已生成的摘要保存，重启后仍可见

### 用户看到什么

- **摘要面板**：在阅读区侧方弹出或覆盖显示
- **流式文本**：内容实时增长，类似 ChatGPT 打字效果
- **状态指示**：生成中显示加载动画，完成/失败显示对应状态

### 交互流程

```
阅读文章 → 点击摘要按钮 → 1 秒防抖 → 开始生成（流式）
                                      → 可取消
                                      → 完成/失败
```

### 行为约束（开发必须遵守）

- **1 秒防抖**：用户点击后等 1 秒，如果 1 秒内再次点击则重新计时
- **串行执行**：同一时间只有一篇摘要在生成
- **latest-only 队列**：如果在生成中用户又触发另一篇，等待队列只保留最新请求
- **不自动重试**：失败后不自动重试，用户手动重新触发
- **不自动取消**：进行中的摘要不会因为切换文章而自动取消

## ④ AI 双语翻译

### 用户能做什么

- **触发翻译**：在阅读文章中点击翻译按钮
- **段落级对照**：每个段落原文在上、译文在下，双语对照
- **并发翻译**：多个段落同时翻译（默认 3 段并发，可调 1~5）
- **重试失败段落**：单个段落翻译失败后，可单独重试该段落
- **清除翻译**：一键清除所有翻译结果，回到原文
- **取消**：翻译进行中可取消

### 用户看到什么

- **双语布局**：大屏左右对照（原文左、译文右），小屏上下对照
- **段落状态**：等待中（灰色）、翻译中（闪烁）、完成（正常）、失败（红色标记 + 重试按钮）
- **工具栏**：翻译进度条、重试全部失败段落、清除翻译、回到原文

### 交互流程

```
阅读文章 → 点击翻译 → 段落切分 → 并发翻译（3 段一组）
                               → 单个失败可重试
                               → 全部完成/取消
                               → 可清除回到原文
```

### 行为约束

- **段落切分**：按 `<p>` `<ul>` `<ol>` 标签切分。文章标题/作者合成为第 0 段（保证双语对照中标题对齐）
- **有界并发**：`Semaphore` 限制 1~5 并发，默认 3
- **持久化**：翻译结果存入数据库，重启后恢复
- **重试**：失败段落单独重试，不影响已完成段落

## ⑤ 笔记与文摘导出

### 用户能做什么

- **写笔记**：在阅读文章中打开笔记编辑器，写 Markdown 笔记
- **保存笔记**：笔记关联到当前文章，自动保存
- **导出单篇**：将当前文章 + 笔记导出为 Markdown 或 HTML 文件
- **导出多篇**：按日期/订阅源筛选 → 批量导出为单个文件

### 用户看到什么

- **笔记编辑器**：简易 Markdown 编辑框（textarea），在阅读区侧方
- **导出面板**：选择格式和保存路径

### 交互流程

```
阅读文章 → 打开笔记 → 编辑 → 保存（自动）
                         → 导出（手动选择格式和路径）
```

## ⑥ 标签系统（v2）

### 用户能做什么

- **手动打标签**：为文章添加一个或多个标签
- **按标签筛选**：在侧边栏或搜索中按标签筛选文章
- **AI 标签建议**：点击"建议标签" → AI 分析文章内容 → 返回推荐标签 → 用户确认添加
- **标签库管理**：合并重复标签、设置别名、清理未使用标签

## ⑦ Token 用量统计（v2）

### 用户能看到什么

- **用量面板**：按 Provider / Model / Agent 维度查看 Token 消耗
- **趋势图表**：每日/每周用量趋势
- **成本估算**：基于模型定价的近似费用

## ⑧ 多语言 UI（v2）

### 用户能做什么

- **切换语言**：设置中选择中文或英文，界面文字即时切换
- **记住选择**：语言选择持久化，重启保持

## 全局交互约定

### 键盘快捷键（Stage 4）

| 快捷键 | 功能 |
|---|---|
| `j` / `k` | 下一条 / 上一条文章 |
| `s` | 触发摘要 |
| `t` | 触发翻译 |
| `n` | 打开笔记 |
| `r` | 刷新当前订阅源 |

### 桌面通知（Stage 4）

- 同步完成后，如有新文章弹出系统通知："{N} 篇新文章"

### 错误处理

- 网络错误：Toast 提示 "网络连接失败"，不阻塞操作
- Feed 解析失败：侧边栏该 Feed 标记警告图标，记录日志
- AI API 错误：面板显示 "请求失败，请检查 Provider 配置"
- 所有错误不崩溃、不弹模态框（除非是致命操作）

## 未明确定义的行为（留给 AI 自行设计）

以下行为 Mercury 原版未明确约束，或我们选择不约束：

- 侧边栏的精确像素宽度（默认 280px，可调整）
- 文章列表的排序方式（默认按发布时间倒序）
- 动画/过渡细节
- 空状态的占位提示文案
- 加载中 spinner 样式
