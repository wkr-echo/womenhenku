## 一、项目初始化阶段（7/12 ~ 7/13）

### 1.1 核心语言：Rust over Swift

- Swift on Windows 工具链不成熟，社区资料少
- Rust 编译器报错详细，编译通过即高概率正确，适合 AI 辅助
- Rust + Tauri 同语言单进程，零 IPC，复杂度低 → **ADR 001**

### 1.2 架构：Tauri 单进程

- Rust 核心直接嵌入 Tauri 2，通过 Tauri Command 暴露接口
- 单进程消除 JSON-RPC IPC 的序列化开销与调试复杂度 → **ADR 003**

### 1.3 AI 协议：OpenAI 兼容

- 行业事实标准，覆盖 DeepSeek/Ollama/vLLM/通义千问/智谱
- v1 不引入自建多协议抽象层 → **ADR 002**

### 1.4 前端：React + TypeScript

- AI 训练数据中 React 代码量远超 Svelte/Vue，生成质量最高 → **ADR 004**

### 1.5 功能优先级

- v1: ①②③④⑤（Feed + 阅读 + 摘要 + 翻译 + 笔记导出）
- v2: ⑥⑦⑧（标签 + 用量统计 + 多语言）

## 三、依赖关系分析（7/13）

### 3.1 唯一硬阻塞：数据库 Schema

- 所有模块都依赖数据库读写
- Schema 不定，后面全在沙上筑塔
- **数据库 Schema 是唯一必须三人一起等的环节**

### 3.2 可并行推进

- Schema 完成后，三条线可用 mock 数据各自独立推进
- 模块 2（AI）依赖 Readability 输出，可在 Readability 未完成前用 mock HTML 独立开发 OpenAI 协议 + SSE 流式
- 模块 3（笔记）依赖 Entry 存在，可用 mock 数据先开发 CRUD 和 UI
- 联调期只有模块 2 需要等 Readability，模块 1/3 不受影响

## 九、实现阶段决策（7/14）

> 核心原则：团队为大一新生，全依赖 AI 辅助。所有决策优先 AI 训练数据量 > 技术先进性。编译器报错必须能直指问题位置，AI 一次修正成功。

### 9.1 Cargo 结构：单体 `src-tauri/`

- Tauri 官方模板默认单体结构，AI 训练数据 100% 覆盖
- 目录隔离（`feed/` `db/` `reader/` `agent/` `digest/`）已满足阶段独立可回滚

### 9.2 SQLite：`rusqlite`（bundled）+ `r2d2`

- Tauri 官方文档推荐，AI 见过的标准模式
- bundled feature 静态编译 SQLite，三平台零额外依赖

### 9.3 复刻定位：功能体验复刻

- 复刻的是用户看到什么、能做什么、交互流程。内部实现完全自行设计

### 9.4 React 技术栈

| 项 | 选择 | 理由 |
|---|---|---|
| CSS | Tailwind CSS | AI 见过最多 |
| 组件库 | shadcn/ui | 代码在项目里，报错在自己文件里 |
| 状态管理 | Context + useReducer | 官方方案，报错简单 |
| i18n | 伪 i18n `t()`（v2 升级 react-i18next） | v1 仅中文 |

### 9.5 Tauri 错误传递：`Result<T, String>`

### 9.6 配置存储：TOML（通用）+ Tauri secure-store（凭证）

### 9.7 开发环境：Rust 1.80+ / Node 20 LTS / Tauri CLI 2.x

### 9.8 推迟决策（Stage 4 再定）

- Markdown 编辑器：先用 `textarea`
- 模板引擎：`handlebars` vs `tera`，选 AI 一次通过率更高者
- 自动更新方案：Tauri `tauri-plugin-updater`
